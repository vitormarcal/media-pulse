# Feature Discovery Workflow

Quero desenvolver uma nova feature.

NÃO escreva código ainda.

Primeiro:

* leia `PRODUCT_CONSTITUTION.md`
* leia `GOVERNANCE.md`
* leia `DESIGN.md` se houver impacto visual/frontend
* leia docs de domínio/API relevantes

Depois:

* faça perguntas para refinar a feature
* faça apenas uma pergunta por vez
* priorize:

    * valor para o owner
    * simplicidade
    * UX
    * domínio
    * escopo mínimo útil
* evite perguntas técnicas prematuras
* questione complexidade desnecessária
* questione overengineering
* ajude a encontrar a menor feature valiosa possível
* evite transformar a feature em plataforma genérica
* preserve alinhamento com a constituição do produto

Importante:

* prefira decisões reversíveis
* prefira heurísticas simples
* prefira usar dados já existentes
* evite novas dependências
* evite nova arquitetura sem necessidade clara

Quando eu disser “ok”:

* produza uma feature spec curta
* objetiva
* sem burocracia
* focada em intenção e escopo

A spec deve conter apenas:

* problema
* objetivo
* escopo
* API/UI
* non-goals
* heurística/regras
* acceptance criteria mínimos

Depois pare e aguarde aprovação.
